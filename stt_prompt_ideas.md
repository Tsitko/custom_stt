Для работы языковой модели нужно задать запрос, в котором будет прописано не только «что» делать, но и «как» именно делать, чтобы избежать неоднозначностей. Ниже – расширенная формулировка, включающая подробные правила, примеры преобразований и правила транскрипции английских терминов.

---

## Инструкция для модели

1. **Общая цель**  
   Преобразуйте исходный текст, полученный от распознавания речи, в чистый, читаемый русский документ.  
   Выполните все действия, перечисленные ниже, и верните окончательный вариант.

2. **Пошаговые правила**  

   1. **Развёртывание сокращений**  
      - Если в тексте встречается аббревиатура, добавьте её полное название в скобках, если это первое упоминание.  
      - При повторном появлении не добавляйте расшифровку.  
      - Сокращения:  
        * «СКД» → «Система контроля доступа»  
        * «API» → «апий»  
        * «CI» → «с‑и»  
        * «LLM» → «элэлэм»  
        * «CPU» → «сипию»  
        * «RAM» → «рэм»  

   2. **Орфографическая коррекция**  
      - Исправляйте типичные ошибки, связанные с фонетическим распознаванием (напр., «статут» → «статус», «найдено» → «найдена»).  
      - Сохраняйте правильный род и число существительных.  

   3. **Пунктуация**  
      - Каждый перечень отделяйте запятыми, завершайте точкой.  
      - После вводных конструкций ставьте запятую (например, «Проверьте наличие сертификата, чтобы обеспечить безопасность»).  
      - Периоды в сложных предложениях разбивайте, если они слишком длинные.  

   4. **Синтаксическая реструктуризация**  
      - Переформулируйте фразы так, чтобы они были однозначными и логичными.  
      - Если предложение содержит несколько действий, разделите их точками с запятой или точками.  

   5. **Транслитерация английских терминов**  
      - Переведите английские слова в русскую транскрипцию:  
        * «CPU» → «сипию»,  
        * «LLM» → «элэлэм»,  
        * «GPU» → «гипи»,  
        * «IoT» → «ио‑т»  
        * «SQL» → «ескьюэл»  

   6. **Числа**  
      - Пишите все цифры словами:  
        * «3» → «три»,  
        * «12» → «двенадцать»,  
        * «2023» → «две тысячи двадцать три»,  
        * «4.56» → «четыре целых пятьдесят шесть сотых».  
      - Для дат использовать формат «два‑тридцать‑сентября‑две тысячи двадцать четыре».  

3. **Примеры преобразований**  
   - *Сырой*: «В сервисе есть API, CI и LLM»  
     *Отредактированный*: «В сервисе есть апий, с‑и и элэлэм, обеспечивающие быстрый доступ к данным».  
   - *Сырой*: «Мы запустили новый скрипт, но возникла ошибка 404»  
     *Отредактированный*: «Мы запустили новый скрипт, но возникла ошибка кода четырёх‑сот пятьдесят четырёх; страница не найдена».  
   - *Сырой*: «Пользователь зарегистрирован, но статус не актив»  
     *Отредактированный*: «Пользователь зарегистрирован, но статус не активен (доступ ограничен)».  

4. **Формат ответа**  
   - Верните только окончательный текст без комментариев, примеров или инструкций.  
   - Соблюдайте правила транскрипции и записи чисел.

Текст: <ТЕКСТ>  
Ответ: <Окончательный текст>»
